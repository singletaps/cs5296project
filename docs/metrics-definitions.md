# 指标与口径（Group 259）

## 端到端延迟（同步 API）

- **起**：客户端发起 HTTP 请求（TLS 握手完成后发送 body 的时刻，以压测工具定义为准；k6 为请求发送开始）。
- **止**：收到 HTTP **200** 且响应 JSON 中 `status === "succeeded"`。
- **报告**：同时给出 p50、p90、p99 与样本量。

## 冷启动（Lambda）

- **定义**：函数实例此前空闲至少 **30 分钟** 后，首个请求的处理路径。
- **对齐 CloudWatch**：记录日志中的 **Init Duration**（若存在）与总延迟对比。
- **注意**：大容器镜像会拉长 init；需在报告中写明镜像 digest 与内存配置。

## 冷启动（EC2）

- 若测量「容器或进程重启后首请求」，须在报告中明确 **不等于** 云主机从停机状态启动，以免与 Proposal「常驻实例」混淆。

## 错误与超时

- **429 / 504**：可能来自 API Gateway 或负载均衡；API Gateway 与 Lambda 同步集成约 **29s** 上限，超长转换在经 API GW 的路径上可能表现为网关超时，需在实验矩阵中分层（短任务同步 vs 长任务异步）。
- **应用错误**：以响应 JSON `error.code` 为准（见 OpenAPI）。

## 吞吐

- **RPS**：压测工具报告的每秒成功请求数（与转换类型、输入 `role` 一起记录）。
